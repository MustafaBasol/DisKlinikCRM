# Landing Page Acceptance Checklist

## Ürün mesajı

- [ ] İlk ekranda bunun diş klinikleri için CRM olduğu anlaşılıyor
- [ ] Randevu, hasta, tedavi, ödeme ve görev yönetimi geçiyor
- [ ] Çoklu klinik yönetimi net şekilde vurgulanıyor
- [ ] Klinik sahibi/persona dili doğru kullanılıyor
- [ ] Abartılı veya kanıtsız iddialar yok

## Tasarım

- [ ] Modern B2B SaaS hissi var
- [ ] Diş/medikal güven hissi veriyor
- [ ] Beyaz alan yeterli
- [ ] Dashboard mockup profesyonel görünüyor
- [ ] CTA butonları görünür
- [ ] Mobil görünüm düzgün
- [ ] Renkler tutarlı

## Teknik

- [ ] Landing componentleri izole
- [ ] Auth/dashboard/backend dosyaları gereksiz değişmedi
- [ ] TypeScript hatası yok
- [ ] Build çalışıyor
- [ ] Lint çalışıyor veya mevcut hatalar raporlandı
- [ ] Dummy data ayrı dosyada
- [ ] Gereksiz paket eklenmedi

## Dönüşüm

- [ ] Demo CTA hero’da var
- [ ] Demo CTA sayfa sonunda tekrar var
- [ ] Form alanları mantıklı
- [ ] Çoklu klinik sahibi için özel bölüm var
- [ ] SSS karar itirazlarını azaltıyor

## Merge hazırlığı

- [ ] Değişen dosyalar listelendi
- [ ] Route bilgisi net
- [ ] Tasarım tokenları mevcut yapıyla uyumlu
- [ ] Sonradan gerçek demo form API’sine bağlanabilir
