## Goal

Improve the visual quality of the Dental Clinic CRM landing page by borrowing design-system thinking from open-design:

- Strong visual hierarchy
- Clean SaaS composition
- Consistent spacing
- Polished dashboard mockups
- Component-based landing sections
- Brand-grade design tokens

## Rules

1. Do not copy large blocks of source code blindly.
2. Do not add open-design as a production dependency unless explicitly required.
3. Do not introduce experimental runtime requirements.
4. Do not alter the existing app architecture.
5. Extract only general design ideas:
   - Layout rhythm
   - Section composition
   - Design token structure
   - Component naming quality
   - Mockup presentation pattern
6. Keep implementation native to the current repo stack.
7. Make sure all generated UI is original and specific to a dental clinic CRM.

## Design system mapping

For this landing page, use a “premium medical SaaS” design system:

- Trustworthy
- Clean
- Calm
- Operational
- Professional
- Slightly premium, not luxury
- Product dashboard led
